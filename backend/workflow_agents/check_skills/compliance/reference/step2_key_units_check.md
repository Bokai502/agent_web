## 判定规则
1. 仅依据输入的要求和检查结果进行判定，不得新增未给出的元器件或质量等级。
2. 对关键部位、关键功能、低等级质量等级、进口/目录外等风险因素应重点说明。
3. 若存在不符合项，结论写“部分符合”或“不符合”；若输入无不符合项，结论写“符合”。
4. 不符合原因必须对应到具体型号规格或具体质量等级，不写空泛判断。

### 问题分析
用 1 段话说明不符合项的共性、涉及范围和潜在影响；如无问题，说明“未发现关键元器件划分不符合项”。

### 整改建议
1. 针对不符合项提出具体、可执行的整改建议。
2. 如无不符合项，提出后续复核和配置管理建议。

## 注意事项
1. 使用正式、客观、规范的技术审查语言。
2. 只输出报告正文，不输出分析过程或字数统计。
3. 正文建议控制在 500 字以内；表格完整性优先于字数限制。
