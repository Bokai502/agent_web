# 42 已知限制与扩展点知识库

这部分是 AIGNC 工具最重要的护栏之一。目的不是列出所有缺点，而是明确：

- 什么不能直接做
- 什么只能近似做
- 什么应该从哪个模板扩展

## 1. 传感器类

### 1.1 无某类传感器时常走真值直通
例如：

- 无星敏时，姿态可能直接给真值
- 无陀螺时，角速度可能直接给真值
- 无 GPS 时，轨道状态可能直接给真值

所以“用户要求真实测量闭环”时，必须显式核查传感器数量与处理链。

### 1.2 FGS 不是完整成像相机
FGS 更像：

- 精跟踪视线偏差传感器

不是：

- 完整二维图像帧相机

但可作为简化焦平面相机模板。

## 2. 执行机构类

### 2.1 当前四轮零空间动量管理不完全通用
当前自定义 CFS_FSW 中的四轮零空间阻尼默认适配当前四轮构型。

结论：

- 动力学支持任意轮构型
- 当前动量管理辅助律不保证自动泛化

### 2.2 帆板展开没有独立现成模块
可以通过：

- `Body + Joint`
- `PASSIVE` 或 `ACTUATED`

实现展开，但复杂机构逻辑通常需扩展 `AdHocJoint()`

## 3. FSW / GNC 类

### 3.1 当前 CFS_FSW 不是完整导航滤波框架
当前更多是：

- 传感器处理
- 模式机
- 控制律

不是完整：

- 姿态确定 + 轨道确定 + 估计融合框架

### 3.2 内置模式与自定义模式要分开看
内置 `THREE_AXIS_FSW`、`SPINNER_FSW` 等和当前自定义 `CFS_FSW` 不是同一实现链。

## 4. 任务级限制

### 4.1 docking / capture
当前没有现成高保真的对接/捕获任务模型。

最接近的是：

- ISS + Shuttle rendezvous / prox ops Demo

### 4.2 lunar GPS
绕月不支持当前地球 GPS 模型。

### 4.3 magnetic control outside Earth missions
月球等任务中，磁控一般不能照搬地球 LEO 思路。

## 5. 扩展模板建议

### 新相机 / 简化焦平面成像
建议模板：

- `FGS`

### 新视线传感器
建议模板：

- `FSS`
- `FGS`

### 新惯导类传感器
建议模板：

- `Gyro`
- `Accelerometer`

### 新内部动量执行机构
建议模板：

- `Wheel`

### 新简单关节机构
建议模板：

- `ActuatedJoint`
- `PassiveJoint`
- `AdHocJoint`

## 6. AIGNC 工具中的决策建议

当用户提出某项需求时，工具应输出三种判断之一：

- `supported`
- `supported_with_assumptions`
- `requires_extension`

不能直接跳过这一步进入配置或代码生成。
